# 月鹿小红书图文生成器（yuelu-xhs）

把月鹿写的 Markdown 文章自动切页排成小红书轮播图。每张图 1242×1656（小红书 3:4），白底黑字 + 复古绿无关，纯 Notion Exporter 同款风格。

## 何时触发

- `/yuelu-xhs <md路径>`
- "把这篇文章转成小红书"
- "把 xxx.md 排成小红书图文"
- "做小红书轮播图 / 做小红书图文 / 做小红书"
- "把月鹿这篇文章排成小红书"

## 工作流

### Step 1：确认输入

- 用户给了 md 路径 → 用它
- 用户没给 → 询问哪个 md 文件，或在 `articles/` 目录里选

### Step 2：检测 AI 配图（关键）

⚠️ **必做这一步**，否则 AI 配图不会出现在小红书图里。

执行：
```bash
ls "<文章所在目录>/配图/插画/<文章 slug>/" 2>/dev/null
```

如果找到这个子目录（比如 `articles/配图/插画/mofang纪实/`），里面通常有：
- `头图.png`
- `配图-01.png` ~ `配图-NN.png`

然后 grep 原文 markdown 检查这些图有没有被 `![]()` 引用：

```bash
grep -E "!\[.*\]\(.*插画.*\)" <md路径>
```

**如果原文没引用**，主动提示用户：
> 我看到 `配图/插画/<文章名>/` 里有 N 张 AI 配图（头图.png + 配图-01~04.png），但原文 markdown 还没引用。要不要先把这些图插入到对应章节？告诉我每张图想放哪个章节标题之下，我帮你加。

让用户决定每张图位置，用 Edit 工具把 `![alt](配图/插画/xxx/yy.png)` 插到对应章节。

如果用户说"不用插"或"已经插了"，直接进 Step 3。

### Step 3：跑脚本

```bash
node ~/.claude/skills/yuelu-xhs/md-to-xhs.mjs <md路径>
```

输出在 `<md同目录>/<slug>-xhs/`：
- `<slug>-preview.html` —— 浏览器预览（所有页并排展示）
- `<slug>-01.png` ~ `<slug>-NN.png` —— 各页 PNG

脚本会打印总页数。

### Step 4：打开预览

```bash
open "<输出目录>/<slug>-preview.html"
```

### Step 5：报告

简明告知：
- 总页数
- 输出目录路径
- 提醒用户在浏览器看效果，如果某页不满意，告诉你要改什么

## 关键约定

| 项 | 值 | 备注 |
|---|---|---|
| 页面尺寸 | 1242 × 1656 | 小红书 3:4 |
| 页脚 | 176px，左下"月鹿造物" + 右下"页码/总数" | 不放小红书号（小红书 App 自带账号水印） |
| 字号 | 正文 34px / 行距 1.6 | |
| 切页方式 | 真浏览器测量 + 贪心切页 | 不靠估算 |
| 图片处理 | 等所有图片完整加载后量真实尺寸 | 不会被裁 |

## 不该做的事

❌ **不要用 AI 生图**做小红书图。月鹿明确否过这条路。
❌ **不要重写文章内容**变成"金句海报"。原文一字不改。
❌ **不要自动填空白**。底部留白是 OK 的，不要自作主张挪段落填空。
❌ **不要改脚本里的字号、页脚高度、SAFE_MARGIN 等参数**，除非用户明确要求。

## 改图

如果用户对某页不满意：
- **改文字** → 改原文 md 对应段落 → 重跑脚本
- **改图位置** → 在 md 里挪 `![]()` 引用 → 重跑脚本
- **不要直接改 PNG**，所有改动通过原文驱动

## 历史踩坑（重要）

- `setContent` + `file://` 协议下图片不会等加载完，必须写临时文件 + `goto` 才能拿到真实图片高度
- 测量容器里**不要**给每个块外包 div，会让 CSS 选择器失效，块测出来都是默认行高
- 测量准确后 SAFE_MARGIN 用 20px 就够，之前因为测量不准用过 80px
